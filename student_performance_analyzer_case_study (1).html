Portfolio Case Study

Problem Statement

Academic institutions lack scalable, automated tools to identify struggling students before they fall irreversibly behind. Manual review of attendance logs and grade sheets is slow, inconsistent, and reactive — not predictive.

Objective

Build a Python-based analytics system that ingests multi-dimensional student data, identifies at-risk individuals through statistical profiling, and delivers visualization-backed insights to support early intervention.

Dataset Overview

  • 200+ student records covering attendance percentage, subject-wise marks, and overall academic standing
  • Multi-subject scope enabling cross-subject performance correlation analysis
  • Raw data ingested via Excel; cleaned and transformed entirely through Pandas pipelines
  • Missing value treatment, normalization, and categorical encoding applied before analysis

Approach / Methodology

  • Data Ingestion: Loaded Excel sheets into Pandas DataFrames; standardized column naming and data types
  • Exploratory Data Analysis: Computed descriptive statistics, identified outliers, and explored attendance-grade correlations
  • At-Risk Segmentation: Defined risk thresholds (attendance below 75%, marks below 40%) and flagged students meeting one or more criteria
  • Subject-Wise Trend Analysis: Aggregated average performance per subject to detect systemic weak areas
  • Visualization Layer: Built bar charts for subject averages, heatmaps for attendance-grade correlation, and distribution plots for score spread
  • Report Generation: Compiled all findings into a structured PDF report with prioritized recommendations

Key Features

  • Automated at-risk student identification using multi-factor threshold logic
  • Subject-level performance heatmap revealing correlation clusters between subjects
  • Attendance vs. marks scatter analysis exposing the attendance-performance relationship
  • Modular, reusable codebase — swappable dataset with no code changes required
  • Exportable visualizations in PNG format for integration into academic reports

Results & Insights

  • Attendance below 70% showed strong negative correlation with final scores, validating its use as a primary risk indicator
  • Mathematics and Physics emerged as the highest-failure subjects, flagged for targeted curriculum support
  • Score distribution analysis revealed a bimodal pattern — suggesting two distinct learner cohorts requiring differentiated strategies
  • Heatmap correlation revealed that poor performance in one core subject predicted underperformance in 2+ others with ~74% accuracy

Interview Q&A

QWhat was the biggest technical challenge you faced?
The messiest part was data cleaning — real attendance records had inconsistent formatting, missing values, and duplicate entries. I used Pandas to build a cleaning pipeline that handled nulls via median imputation, standardized date formats, and flagged anomalies. That process taught me that 60% of analytics work is just getting data into a shape you can actually trust.
QHow did you decide which students to flag as "at-risk"?
I used a multi-threshold logic: any student with attendance below 75% OR subject score below 40% was flagged. I then layered severity — students meeting both conditions were prioritized. I validated these thresholds against the overall score distribution to ensure they weren't too aggressive or too lenient. It's a rule-based approach, but it mirrors how academic early-warning systems work in practice before ML is introduced.
QHow would you scale this if given 10,000 records instead of 200?
Pandas handles up to a few million rows comfortably, so 10K is no issue. Beyond that, I'd move to Dask for parallel processing or push the data into a SQL database and run aggregations there. I'd also replace the static report with a live Streamlit dashboard — so educators could filter by class, subject, or risk level in real time instead of waiting for a new report to be generated.
